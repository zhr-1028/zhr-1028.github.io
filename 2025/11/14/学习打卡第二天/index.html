JAVA（OOP）回顾

1.（方法/构造函数)的重载

OverLoading： 同一个类中定义了多个方法名相同而参数不同的方法，重载在同一个类中，方法名相同，参数不同（参数的个数、顺序、类型不同）。

2.this关键字

this是一种特殊的引用，指向当前对象，一般有以下两种用法
—如果发生局部变量和成员变量命名冲突时，可以通过this.成员变量名的方式区分成员变量和局部变量。
—一个构造方法中需要调用本类的另一个构造方法，可以通过this()的方式调用，但this()必须要书写在第一行。

3.final关键字

final 关键字用来修饰类、方法和变量，具有以下特点：
—修饰符方法：方法不能重写
—修饰类：类不能继承，final 类中的所有成员方法都会被隐式的指定为 final 方法
—修饰符变量：该变量为常量，如果是引用类型的变量，则在对其初始化之后便不能让其指向另一个对象。

5.继承与多态

—Java只支持单继承，不支持多继承。一个类只能有一个父类，不可以有多个父类。Java支持多层继承(继承体系)。Java继承了父类非私有的成员变量和成员方法，但是请注意：子类是无法继承父类的构造方法的。
子类从父类继承的某个实例方法无法满足子类的功能需要时，需要在子类中对该实例方法进行重新实现，这样的过程称为重写，也叫做覆写、覆盖。
—多态：一种事物的多种形态（多态的前提：继承、重写、向上转型），通过多态可以提高代码的可重用性，降低模块之间的耦合度

抽象类的核心特点（通过案例体现）
不能实例化：Vehicle 是抽象类，无法直接创建对象（如 new Vehicle(…) 会报错），只能通过子类实例化。
包含抽象方法：run() 被 abstract 修饰，没有方法体，强制子类必须实现。（否则子类也需声明为抽象类）。
可包含普通方法：start()、stop() 是普通方法，提供通用实现，子类可直接使用或重写。
作为父类体现多态：父类引用（Vehicle car）指向子类对象（new Car(…)），调用 run() 时执行子类的实现，体现多态特性。

7.接口

接口的核心特点:
—接口中的方法默认是 public abstract（可省略不写），只有声明，没有方法体。
—接口中的变量默认是 public static final（常量），必须初始化，且无法修改。
—类通过 implements 关键字实现接口，必须实现接口中所有抽象方法（否则该类需声明为抽象类）。
—一个类可以实现多个接口（弥补 Java 单继承的限制），接口之间也可以通过 extends 继承（支持多继承）
—接口是 Java 中实现多态和代码解耦的重要手段，广泛用于框架设计

接口VS抽象类
设计目的：
定义功能规范（“能做什么”）VS 定义类的模板（“是什么”+“能做什么”）
继承 / 实现 :
类可实现多个接口，接口可多继承接口 VS 类只能单继承抽象类
